# <center>vscode 插件：picgo</center>

## 前言

一直都是用 obsidian 写 md 文档，本地复制图片可以直接显示，还是挺方便的，但是如果要上传到 github 上面，文件地址因为是本地的，无法正常显示。然后得去 github 上面调整，很不方便。

## 图床

搜索了几款图床工具，还是 vscode 的 picgo 方便。
只需配置好 github 的相关信息，复制好图片，然后 ctrl + alt + c，即可上传到 github 仓库中，又设置了 cdn 地址，图片加载也很快

## raw.githubusercontent.com 不能访问的问题

部署到服务器上，发现`cdn.jsdelivr.net`的图片会重定向到`raw.githubusercontent.com`，`raw.githubusercontent.com`访问不了

到[查询网](https://site.ip138.com/raw.githubusercontent.com/)，查下`raw.githubusercontent.com`的解析 ip

然后打开 hosts ，加上

```js
185.199.111.133 raw.githubusercontent.com
```

就可以正常访问图片了

## 总结

配置完 picgo 的 github 信息后，现在写 md，速度真是快的飞起，再也不用担心图片上传的问题了
